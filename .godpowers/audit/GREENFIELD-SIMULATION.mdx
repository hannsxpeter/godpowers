# Greenfield Simulation Audit

## Simulation Result

- [DECISION] A new maintainer should encounter one front door, a small core profile, explicit proof scope, and disk-derived status before the 122-command reference surface.
- [DECISION] The current architecture supports that path through `/god`, `/god-first-run`, `/god-demo`, `/god-status`, and profile expansion.
- [DECISION] The full command surface remains available through `builder`, `maintainer`, `suite`, and `full` profiles.

## Friction Found

- [DECISION] Fixture-backed Quick Proof could be mistaken for evidence about the selected project.
- [DECISION] A default profile near 20 commands asked a first-time user to distinguish implementation leaves before learning the front door.
- [DECISION] Stale always-loaded context could give a new agent obsolete version and surface facts.

## Greenfield Standard

- [DECISION] The safe first proof performs no writes and states the fixture boundary in output.
- [DECISION] Current-project proof requires the explicit read-only `--inspect-project` flag.
- [DECISION] The core profile exposes high-level verbs and preserves expansion paths for all compatibility commands.
