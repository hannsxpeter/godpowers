# Phase 4 State Read Inventory

## Scope

- [DECISION] This inventory covers root `SKILL.md`, `lib/`, `skills/`, `routing/`, `workflows/`, `scripts/`, tests, and docs because route generators and workflow YAML create state reads even when the checked-in route files are the runtime surface.
- [DECISION] The explicit Phase 4 target paths are `.godpowers/PROGRESS.mdx` and Godpowers-owned per-tier `STATE.md` files.
- [DECISION] Suite-level `.godpowers/suite/STATE.mdx` and workstream-local `.godpowers/workstreams/<name>/PROGRESS.mdx` are recorded as adjacent state surfaces, but they are not the first migration target for root project state authority.
- [DECISION] Legacy planning `.planning/STATE.md` is recorded as a legacy-source exception because Phase 4 explicitly keeps imported planning systems as source-system reads.

## Classification

- [DECISION] `decision-read` means a command, route, agent, gate, or test uses markdown state existence or content to decide whether work may proceed, which command runs next, or whether an artifact passes.
- [DECISION] `display-read` means markdown state is used only to render human context, summaries, examples, file trees, or historical reports.
- [DECISION] `migration-read` means markdown state is read only to import, sync, archive, or generate a replacement representation.
- [DECISION] `legacy-source-read` means an external planning system owns the markdown file and Godpowers must not treat it as authoritative Godpowers state.

## State Schema Gaps

- [DECISION] The initialized-project predicate gap is closed by computed `state:initialized == true`, which treats project identity plus `tiers` in `state.json` as the route authority.
- [DECISION] Generated root and per-tier markdown views now carry path-specific checksums inside managed fences; structured `state.json` view metadata for generated-at, source-version, and dirty-view warning state remains deferred.
- [DECISION] Completed: `state.json` schema now documents build verification evidence for exact command, status, exit code, run time, duration, and failure diagnostics.
- [DECISION] `state.json` still needs richer design state evidence for design-review verdict, PRODUCT.md presence, generated-view checksum, and bridge command history beyond the complete-status gate check.
- [DECISION] Completed: `state.json` schema now documents deploy, observe, and launch evidence fields for readiness scope, verified target type, target evidence, external access deferral, waiting artifacts, rollback evidence, SLO evidence, alert evidence, runbook evidence, and launch evidence.
- [DECISION] `state.json` needs migration metadata for imported source systems so `.planning/STATE.md` remains an allowed legacy-source read instead of a Godpowers authority read.
